# 变更记录（前端对接向）

> 面向前端/接入方，只记**会影响接口调用的变更**：key 改名、字段/结构变化、规则语义变化等。
> 每次升级请按「迁移指引」逐项自查。key/字段的完整现状始终以 `registry/*.yaml` 与 `GET /api/keys` 为准。

版本对应关系：**1.0.0** = commit `e171ba8`（基线）；**1.0.1** = commit `6892c4d`；**1.0.2** = tag `v1.0.2`（当前对外发布版）；**1.0.3-dev** = branch `v1.0.3`（基于 `v1.0.2` 之后的开发线，未发布）。

## 1.0.3-dev（2026-07-17，开发中）

> 本节汇总 `v1.0.2` 之后、当前 `v1.0.3` 开发分支上的变更。此前草稿中的 `1.0.4`、`1.0.6` 版本号不再作为对外版本使用，相关内容全部收敛到 `1.0.3-dev`。

### 接口与寻址契约

- 三个取数接口仍保持不变：`/api/value`、`/api/records`、`/api/series`。
- registry 内部唯一身份从单纯 `key` 调整为 `(表, key)`；跨表可以复用同一个业务 key。现有全局唯一 key 调用保持兼容。
- `GET /api/keys` 新增 `qualified_key`，格式为 `<表>:<key>`，用于前端列表主键/缓存主键。
- 内部引用需要消歧时使用结构化 `{表, key}`，不提供简写语法。
- `/api/value`、`/api/records`、`/api/series` 均支持按同表父系前缀展开。比如传 `船舶捕捞【产量信息】`，返回该前缀下全部 A 表叶子 key；传 `船舶信息模型`，返回该前缀下全部 C 表叶子 key。
- 注册配置不再维护 `子`、`指标`、`分组` 这类父子元数据；每一行可查询数据都作为一条完整 leaf key 声明。旧的父 key 查询方式通过“父系前缀动态展开”继续兼容。
- 当前 registry 从 `v1.0.2` 的 55 个 key 扩展到 662 个 key（A244 / B8 / C410）；其中新增 628 个、移除 21 个旧 C 表分组容器 key。
- `/api/series` 的 `window` 支持两种写法：全局 `{start,end,points}` 或逐 key `{key:{start,end,points}}`。两种写法不能混用。
- C 表 `points` 缺省统一使用全局默认值 `20`；不再支持每个 key 单独配置 `默认点数`。
- 响应中新增可选字段 `显示`，供前端展示用；旧前端若忽略未知字段，不需要改代码。
- 请求 deprecated/alias key 时，响应顶层可能新增 `warnings`；`GET /api/keys` 会透出 `alias_of`、`deprecated`、`replaced_by`、`remove_after` 等迁移元信息。
- 配置和响应契约中移除无消费价值的 `备注`、`说明` 字段。

### 前端影响

- 旧的精确 leaf key 调用保持可用。
- 旧的父 key 调用保持可用，但现在返回该父系前缀下的 leaf key 数据，不再返回 `子` 或 `指标` 元信息。比如 `船舶.海况` 会返回 `船舶.海况.*` 的各个叶子 key，而不是一个分组说明对象。
- `/api/series` 新增全局 window 写法，逐 key window 写法仍兼容。
- 新增 `显示` 字段、删除内部配置里的 `备注/说明` 字段，不影响按 `value`、`单位`、`values` 解析的旧页面。
- 对接三方原文口径页面时，优先使用 `船舶信息模型`、`天气、海况、水文气象信息`、`虾粉生产线【主要设备及运行状态参数】`、`虾油提取【主要设备及运行状态参数】` 等父系前缀整组取数；需要单项时再使用具体叶子 key。
- 如果前端将 `/api/keys` 结果作为 React key、缓存 key 或配置主键，请改用 `qualified_key`；普通取数请求不需要改。

### 迁移指引（从 v1.0.2 到当前 v1.0.3-dev）

1. 如果前端仍把 `船舶.海况`、`工厂.能耗`、`工厂.虾油提取生产线.生产数据` 等旧分组 key 当成“返回子列表/指标 map”的接口使用，请改为按 `data` 中展开后的叶子 key 渲染；key 发现改用 `GET /api/keys`。
2. 区间曲线同一时间窗优先改用全局 `window: {start,end,points}`，避免给每个 key 重复写同样窗口。
3. 兼容旧 key 时注意读取顶层 `warnings`，并按 `replaced_by` 迁移；业务数据仍在 `data` 内，warning 不会混进 value。
4. 新页面若按三方文档章节展示，直接传三方原文父系前缀，例如 `虾油提取【主要设备及运行状态参数】` 或 `蛋白肽生产线（生产数据）`。

## 1.0.2（2026-07-08）

端口映射调整 + 渔季主数据扩展。**不含 key/字段结构变更，前端一般无需改代码，仅需核对联调地址。**

### 🔧 变更内容

- **服务端口映射调整**（`compose.yaml`）：宿主机对外端口整体改为 6630–6632 段，容器内端口不变。
  - API：宿主 `8000` → **`6632`**（容器仍 `8000`）。联调 Base URL 与 Swagger 改用 `http://<host>:6632`。
  - RTSP：宿主 `8554` → **`6631`**；HLS（浏览器大屏）：宿主 `8888` → **`6630`**。
  - `config/前端使用手册.md` 内示例地址已同步更新为 `localhost:6632`。
- **渔季接口改为返回「当前渔季」**（`app/resolver.py` + `registry/主数据.yaml`）：`GET /api/value` 查询 `渔季` 时，后端按当前业务时间（`APP_TZ`）从渔季表中选出**所处的那个渔季并返回单对象** `{名称, 开始, 结束}`，**不再返回整张区间列表**。
  - 渔季表本身扩为 **2024/25–2030/31 共 7 个渔季**（每个 12-01 ~ 次年 11-30，CCAMLR 口径），仅作为后端选择当前渔季的依据；前端无需感知全表。
  - **前端影响**：原先解析 `value` 为数组的代码需改为**按单对象读取** `value.名称 / value.开始 / value.结束`。

### 迁移指引（前端自查清单）

1. 把联调/本地的 API 地址端口从 `8000` 改为 `6632`（生产/联调环境以运维给出的地址为准）。
2. 大屏/视频若直连 RTSP、HLS，端口分别改为 `6631`、`6630`。
3. 查询 `渔季` 的界面：把响应 `value` 由「数组」改为「单对象」读取（`value.名称/开始/结束`）；调用方式不变，后端已按当前时间返回当前渔季。

#### 2) `设备.台账`（B 表）结构重构：由「按产线一条」改为「按设备一条」

- **过滤字段**：`filter` 由 `[所属产线]` 扩为 `[所属产线, 设备名称]`——现在可精确到单台设备。
- **响应结构变了**：
  - 旧：每个 `所属产线` 一条记录，`value` 是**该产线的设备数组**。
  - 新：**每台设备一条记录**，`value` 是**单个设备对象**。按某产线查会返回多条（每设备一条），`value` 不再是数组。
- **字段变化**：
  - `所属工艺流程` → 重命名为 `所属产线`。
  - **移除** `当前状态`、`当前温度`（台账不再带运行态）。
  - **新增** `设计参数`（铭牌/设计参数；当前为示例占位，待真实数据校准）。
- **设备实时状态改从 C 表取**：需要设备"当前状态"（运行中/待机/故障…）的，改查对应的 C 离散状态 key，例如 `船舶.虾肉生产线.平板速冻机.状态`、`船舶.虾粉生产线.直接蒸煮器.状态`、`船舶.桁杆泵吸系统.拖网绞车.状态`（用 `/api/series`）。

#### 3) 溯源链设备信息字段重命名：`逆流提取机组` → `逆流提取`

`溯源`（B 表）中「虾油提取生产线」环节的 `设备信息` 下，键名 `逆流提取机组` 统一改为 `逆流提取`。若前端展示溯源设备信息、按该键取值，需同步。

### 🔧 其他变更（一般不影响前端）

- 新增本手册族文档：`config/前端使用手册.md`（接口用法 + 参数说明）。
- `overrides` 演示示例修正为覆盖 C 状态 key（B 表静态覆盖本就不生效）——不影响正常取数。
- Swagger `/api/value` 内置示例 key 更新为新名（原示例用的是旧名，会报未注册）。
- 仓库清理 `.DS_Store`、补 `.gitignore`——与接口无关。

### 迁移指引（前端自查清单）

1. 全局搜索并替换请求代码中的旧 key 前缀（见上表 6 项）。
2. `设备.台账`：如需按设备过滤，`filter` 加 `设备名称`；解析响应时把 `value` 从「数组」改为「单对象」，并注意条目数变多；字段 `所属工艺流程`→`所属产线`，不再有 `当前状态`/`当前温度`。
