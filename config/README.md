# config · 配置注册表（机器可加载层）

本目录是《配置与取数契约》的落地，是 resolver 启动时加载进内存的全部配置。**改数据/指标只动这里，不动框架代码。**

```
config/
├── registry/            # 注册表：每个 key 一条声明（按业务域分文件）
│   ├── 船舶.yaml        # A:信息/规格/设计能力  C:航行海况/捕捞/三线读数/各舱温度/能耗
│   ├── 工厂.yaml        # A:信息/设计能力  C:天气/两线工艺/生产数据(累计)/能耗/库存
│   ├── 主数据.yaml      # A/B:仓储/人员/设备台账/专利/商品/监控清单/渔季
│   └── 溯源.yaml        # B:溯源(环节顺序+条目)/检测
├── constraints.yaml     # 跨 key 约束/派生层（INV-1..6：物料守恒/得率/谱系/温度链/上界/成分）
├── overrides.yaml       # 覆盖/冻结/兜底（剧情注入、历史钉死、真值源兜底）
└── sources.yaml         # 真值源连接（open-meteo / TimescaleDB / MediaMTX；密钥走环境变量）
```

## 约定（详见《配置与取数契约》§0、§5）
- 每条声明含：`key` / `表(A|B|C)` / `成熟度(人工固定|规则生成|真值采集)`，再按表/成熟度补字段。
- **表**决定请求/响应的轴；**成熟度**决定值从哪来（二者正交）。
- C 的 `规则` 参数为**预设标定值**，待演示效果/真实频率调整（见《待确认清单》Q6）。
- 真值采集 key（航行/海况/天气/监控）当前**待对接**（Q3/Q5）；未对接前航行海况走 `fallback` 规则。
- 含 `{...}` 花括号枚举或 `说明:` 的条目是**待逐项展开的占位**（如三线同构项、工段子指标），开发时按同构模板补全。
