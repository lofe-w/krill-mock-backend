# config · 配置注册表（机器可加载层）

本目录是《配置与取数契约》的落地，是 resolver 启动时加载进内存的全部配置。**改数据项只动这里，不动框架代码。**

```
config/
├── registry/            # 注册表：每个 key 一条声明（按业务域分文件）
│   ├── 船舶.yaml        # A:信息/规格/设计能力  C:航行海况/捕捞/三线读数/各舱温度/能耗
│   ├── 工厂.yaml        # A:信息/设计能力  C:天气/两线工艺/生产数据(累计)/能耗/库存
│   ├── 主数据.yaml      # A/B:仓储/人员/设备台账/专利/商品/监控清单/渔季
│   └── 溯源.yaml        # B:溯源(环节顺序+条目)/检测
├── constraints.yaml     # 跨 key 约束/派生层（INV-1..6：物料守恒/得率/谱系/温度链/上界/成分）
├── overrides.yaml       # 覆盖/冻结/兜底（剧情注入、历史钉死、真值源兜底）
└── sources.yaml         # 真值源连接（open-meteo / TimescaleDB / MediaMTX；密钥走环境变量）
```

## 约定（详见《配置与取数契约》§0、§5）
- 每条声明含：`key` / `表(A|B|C)` / `成熟度(人工固定|规则生成|真值采集)`，再按表/成熟度补字段。
- **表**决定请求/响应的轴；**成熟度**决定值从哪来（二者正交）。
- C 的 `规则` 参数为**预设标定值**，待演示效果/真实频率调整（见《待确认清单》Q6）。
- 真值采集 key（航行/海况/天气/监控）当前**待对接**（Q3/Q5）；未对接前航行海况走 `fallback` 规则。
- 含 `{...}` 花括号枚举的条目是**待逐项展开的占位**（如三线同构项、工段子项），开发时按同构模板补全。

## 数据真实度标定（关键：哪些有真实依据、哪些是虚构）

部分参数已用**金蝶 ERP 真实生产数据**标定（来源：mozi-int 仓库对"连云港深蓝生物磷虾油厂"的金蝶只读探测，2026-06，正是本项目甲方的工厂虾油提取生产线）。务必区分：

**✅ 金蝶真实标定（可信，敢用）**：
- 成品**磷虾油成分**：磷脂50-67(典型60)、EPA15-17、DHA7-8.5、虾青素180-312mg/kg、酸价7-8、过氧化值≈0.03、水分≈1.2（注意虾青素单位 mg/kg、酸价 mgKOH/g）。
- **脱脂虾粉(CP001)**：脂肪3.2、蛋白76.6、水分3.2、盐2.76、灰11.5、净含量20kg。
- **全脂虾粉(原料磷虾粉AM001)**：粗脂肪21-23、粗蛋白55、水分5.9-7.4、灰9-12（修正了此前粗脂肪8.3%的错值）。
- **出油率≈18%**(p10/p50/p90=14.7/18/23.2，范围13.4-27)，BOM 5:1，联产上清液油(170/500/异常2000kg)+脱脂虾粉。
- **保质期 730天(24个月)**；库存量级（半成品油≈20000kg/22批、原料粉单批可达33万kg占压）。
- **批次/单号格式**：工单 SCDD-YYMMDD-NNNN；批次 {YYMM}{AKO/ASO/AKP}{序}；质检 FQC/IQC；物料编码 1/2/3* 语义。
- **不合格率极低(≈0.21%)**；活性成分(EPA/磷脂)呈**月度台阶式漂移**(非平稳，campaign 投产)，逼真造数应据此而非平稳白噪声。

**⚠️ 金蝶无、保持合理虚构（不可冒充真实，待真实源再校准）**：
- **工艺过程参数**：乙醇浓度、提取/蒸发/醇沉温度、MVR真空度、酶解PH/温度/时间等——金蝶仅有工艺路线、无过程参数。
- **蛋白肽生产线**：该客户无此线，全部成分/参数虚构。
- **船载三线(虾粉/冻虾/虾肉)**：蒸煮/速冻温度、虾壳残留率、船上产量——陆基厂无。
- **捕捞**：产量/拖网泵吸参数——客户捕捞系统未接入，金蝶溯源止于磷虾粉进厂批次；三方文档“3. 船舶捕捞”已按文档值配置，文档缺失的单位/取值范围/典型值按行业经验补足。
- **冷库温度具体℃**：未发现真实值。

> 一句话：**成品成分/得率/批次/保质期/库存**敢按真实用；**工艺参数/蛋白肽/船载/捕捞**是虚构占位，标注在此，切勿当真实。
