## 1. 项目概览

**背景**："深蓝号"是南极磷虾远洋捕捞加工船，连云港工厂是其岸基精加工基地。甲方要用一块**可视化大屏**展示磷虾从"捕捞 → 船载加工 → 冷链 → 岸基仓储 → 陆地精加工（虾油/蛋白肽）→ 产品溯源"的全产业链运行。真实传感器尚未就位，大屏数据**以假数据为主、少量接真实源**。本项目是这块大屏的**数据后端**。

**定位（要害）**：本系统的价值不在"做一台多快多通用的供数机器"，而在**用一个最小的、与实现无关的、正确的模型，把大屏需要的数据语义一次性表达对**——使它现在能驱动大屏、将来能无痛迁移到真实后端、且不在前端呈现错误/矛盾/将来实现不了的数据。

**三条建模纪律**：
1. 按**本质**建模，对**实现免疫**——模型只描述"数据本质是什么形状"，绝不按"值怎么造"（真值/规则/采集）组织。
2. **模型是源头**，造数引擎与接口都是其可丢弃的下游投影。
3. 与**真实后端对齐**——每个概念都能对上未来真实 MES/溯源系统，这是"减少前端改造"的兑现方式。

## 4. 关键决策记录

1. **按"两根正交轴(选择 × 时间)→ 四个角"建模，实例化为三表**（domain.md §2）：
   - A `(key,value)` 单一站立；B `(key,filter,value)` 可选站立；C `(key,time,value)` 时序。
   - 第四角 D `(key,filter,time,value)` **当前无消费 → 不实例化，记为潜在扩展**（据前端接口查证：溯源只返批次编号、冷库温度按舱、生产数据按窗口累计，无"开放实例集×时间"消费）。
2. **时间是独立的轴，不是 filter 字段**；key 编码封闭已知集，filter 仅用于开放数据驱动集（如产品批号/仓库类别）。**形状由消费决定，不由数据是否存在决定**。
3. **溯源 = 谱系投影，value = `{类别, 溯源链条}`**（domain.md §5，以甲方 6.x 为权威）：链条 = 各环节 `{批次编号 + 本环节数据}`；**追溯=捕捞环节数据、检验=检测环节数据，都在链条内**（三方接口把它俩拍平成顶层字段，不跟）；过程数据**引用不复制**；7 类按路径长短自然区分。
4. **数据成熟度（真值采集/规则生成/人工固定）与形状正交**，是实现旁注，将来整体消失，**不作模型骨架**。
5. **可复现**：规则生成的值由 `(key,网格时间)` 哈希播种的纯函数确定，历史任意时刻查询一致（详见《造数引擎》）。
6. **关注点分离**：后端只给数据；单位拼接、图表结构归前端。接口是最上层投影，**待模型定稿后再细化**。
7. **暂不做**：鉴权/分页/admin/展示层。

## 5. 可配置性契约（改 yaml vs 改代码）★核心卖点

本系统能打的关键：**数据靠配置、语义靠代码**。判据一句话——
前端要的是"**换个数据长相**"（结构/字段/取值）→ 改 **yaml**；要"**系统帮它算/解释点新东西**"→ 改代码。

**✅ 纯 yaml，不动服务代码（resolver/api）：**
- **A/B 表 value 的内部结构任意 reshape**（对象/数组/嵌套/`{环节,数据}`…）——resolver 对 A/B value 是**原样透传**，不解析其内部。
- 增删改 key、增删 B 的条目（造更多数据）、改 filter 取值、调 C 规则参数、切换成熟度。

**⚠️ 要改代码（小而罕见的"原语集"）：**
- 新增一种**表/形状**（超出 A/B/C，如启用 D）；新增一种 C 的**生成 kind**（超出 有界瞬时/累计/离散状态/航迹）；
- 新增/删除一种**被解释的语义**（派生、约束等需 resolver 动手算的）；新增一种**真值源 provider 类型**。

**两个诚实注脚（来自实践教训）：**
1. **测试 `smoke.py` 会随数据结构变**——断言里编码了预期形状；结构变，断言跟着变。这是**测试代码**（正常），不是**服务代码**。
2. **提交卫生**：把"纯配置/结构"改动与"代码/语义"改动**分开提交**。曾把"溯源改结构(配置)"和"删引用(代码语义)"塞进同一 commit，导致"改结构是否动代码"看不清——边界要可审计。

## 6. 维护规则

- **domain.md 是唯一事实来源**：任何最终设计落在 domain.md；《造数引擎》只记实现。
- **改了设计先改 domain.md，再改代码**。
- 文档交叉引用用"标题"而非文件名。提交信息用中文，说明"改了什么 + 为什么"。
- 新增/重组文档后，同步更新本文件第 3 节。
