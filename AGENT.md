## 1. 项目概览

### 1.1 背景

"深蓝号"是南极磷虾远洋捕捞加工船，连云港工厂是其岸基精加工基地。甲方希望用一块**可视化大屏**展示南极磷虾从"捕捞 → 船载加工 → 冷链 → 岸基仓储 → 陆地精加工（虾油/蛋白肽）→ 产品溯源"的全产业链运行情况。

由于真实传感器/系统尚未全部就位，大屏所需数据**以 Mock（假数据）为主**，少量接真实数据源。本项目即这块大屏的**数据后端**。

### 1.2 定位

一个**配置驱动、确定性的数据供给后端**：

- 绝大多数数据是**假数据**——由配置规则即时生成或由配置直接给出；
- 少量数据接**真实来源**（如工厂天气、船舶 TimescaleDB、监控视频流）；
- 对前端提供**统一、通用、简单**的查询接口。

### 1.3 目标

- **简单、快**：能快速实现、快速满足当前大屏需求。
- **通用、灵活、可配置**：用一套通用框架 + YAML 配置覆盖所有数据；**配置即生效**，增删改数据/指标不动框架代码。
- **可复现**：假数据虽是"造"的，但同一时刻的值由规则确定，**历史任意时间查询结果一致**（便于演示、截图、对账）。

### 1.4 核心原则

1. **数据三分类**（按"值怎么来"划分，见 §4）：静态固定值 / 动态规则生成 / 特殊来源。
2. **统一的 `key | filter | value` 契约**：用 `key` 选模块、`filter` 过滤、`value` 原样返回 JSON。配置内容即数据模型即接口出入参。
3. **懒计算 / 规则即真相**：动态数据不落库、即时按规则算，确定性、可复现。
4. **关注点分离**：后端只产出数据本身，单位拼接、图表结构等展示逻辑归前端。
5. **真实数据才落地**：只有接真实来源的数据才涉及存储/外部连接；其余无状态、无数据库。

> 技术栈：Python 3.11+ / FastAPI；动态/静态数据无状态、无数据库；特殊来源按需接 SQLite / TimescaleDB / RTSP(MediaMTX) / LLM 等。

## 4. 关键决策记录（ADR 速查）

> 已与甲方/前端/架构师讨论后拍板。除非有充分理由，勿推翻重议。最终以 `domain.md` 为准。

1. **数据按"值怎么来"分三类**：
   - **① 静态固定值**：人工在 YAML 配置的固定数据（溯源、仓库、监控地址、人员列表、船舶信息等）。配置即数据。
   - **② 动态规则生成**：按 YAML 配置的规则即时生成、可查历史、稳定可复现（温度、浓度、产量、能耗、设备状态等）。配置 `key+filter+值类型+单位+网格(cron)+规则`，不配具体时间/值。
   - **③ 特殊来源**：需特殊支持的（直接接三方接口、接三方并落库、RTSP 视频流、调用 LLM 等）。接口实现与数量视情况而定。
2. **统一 `key | filter | value` 契约**：`key` 选模块/实体，`filter` 是 k-v JSON（支持 {} 不过滤、单/多选、字符串/数值/时间过滤），`value` 为 JSON 原样返回。**接口具体形态（端点数量、动态数据的时间/范围表达、value 形态）在 domain.md 定稿。**
3. **懒计算 / 规则即真相**：动态数据不落库、不缓存、即时按规则算；同一 `(key, filter)` 恒定可复现。
4. **无数据库 / 无状态**：仅"特殊来源"涉及存储/外部连接；静态与动态数据全程无状态。
5. **可复现的实现原理**：种子化纯函数 + 相干噪声（平滑、缓慢、有波动性）；指标间硬约束用"指标依赖图 + 按构造派生"（详见过程文档《造数引擎》，结论将进 domain.md）。
6. **关注点分离**：后端只给数据；单位拼接、ECharts 结构等归前端。
7. **真实数据源已知项**：工厂天气=open-meteo（落 SQLite）；船舶航行/天气=远程 TimescaleDB（透传）；监控=RTSP（MediaMTX 循环推流）。
8. **暂不做**：admin/控制接口、前端兼容的"组合模板层"、后端展示层、鉴权/分页。
9. **编号规则**：预设占位，待业务方提供真实规则后调整。

## 5. 约定与维护规则

- **目录结构**：`docs/01-requirements/`（需求·只读原始件）、`docs/02-design/`（设计）。
- **domain.md 是唯一事实来源**：任何最终设计落在 domain.md；其余设计文档是过程记录，可不再维护。
- **改了设计就改文档**：架构/接口/规则变更，先更新 domain.md，再改代码。
- **文档内交叉引用用"标题"而非文件名**，便于改名不破链。
- **提交信息**：中文，说明"改了什么 + 为什么"。
- **本文件（AGENT.md）是索引的唯一真源**：新增/重组文档后，同步更新第 3 节。
