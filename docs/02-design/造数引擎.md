## 1. 背景与目标

主架构决定：`computed` 型指标的历史数据**不落库**，查询时即时计算。这要求生成函数同时满足三件看似矛盾的事：

1. **可复现 / 稳定**：同一 `(指标, 时刻)` 任何时候、任何进程、任何调用次序，结果完全一致。
2. **逼真**：曲线平滑、缓慢起伏、有多尺度波动性，像真实传感器读数，而非逐点乱跳。
3. **守规则**：满足"温度 25±2℃""缓慢增长的波动""累计量单调""航向航速与坐标一致"等业务约束。

本文档描述实现这三点的引擎原理与实现细节。

### 1.1 核心命题

> 朴素做法 `value(t) = 基线 + 幅度 × random()`（每个点独立摇随机数）能满足"可复现"（只要 random 由 `(指标,t)` 播种），但**满足不了"逼真"**——结果是白噪声，温度会在相邻点之间剧烈乱跳。

引擎的全部技巧在于：把"逐点独立随机"替换为**相干噪声（coherent noise）**——只在稀疏锚点上取随机值，锚点之间平滑插值。由此同时拿到可复现与逼真。

### 1.2 两个正交的机制

| 目标 | 机制 | 所在章节 |
|---|---|---|
| 可复现 / 稳定 | 种子化纯函数（哈希播种） | 第 2 章 |
| 逼真（平滑、缓慢、有波动性） | 相干噪声 + 分形叠加（fBm） | 第 3 章 |

二者组合，再叠加趋势/季节包络与各类规则原语，覆盖文档中的全部数据形态。

## 2. 可复现的根基：种子化纯函数

随机性**不取自系统 RNG**，而取自对 `(指标 key, 锚点索引, 通道)` 的哈希。

```python
import hashlib

def rand_unit(key: str, index: int, channel: str = "") -> float:
    """确定性地返回 [-1, 1) 区间的一个值。
    只依赖入参 → 同输入永远同输出。"""
    h = hashlib.sha256(f"{key}|{channel}|{index}".encode()).digest()
    u = int.from_bytes(h[:8], "big") / 2**64   # → [0, 1)
    return 2.0 * u - 1.0
```

要点：

- **无外部依赖**：不读 `now()`、不读全局状态、不依赖调用顺序。`rand_unit(k, i)` 是数学意义上的纯函数。这是"今天查与明天查同一段历史逐点一致"的根本保证。
- **指标隔离**：`key` 含指标全限定名（如 `磷虾船.海水温度`），不同指标的随机流互相独立、不串扰。
- **多通道**：一个指标常需多路独立随机（如位置的经度/纬度、fBm 的不同倍频），用 `channel` 加盐区分，互不相关。
- **可换实现**：`sha256` 可替换为更快的 `xxhash`；`int → float` 后可再喂入 PCG/SplitMix64 等 PRNG 以改善分布。接口不变。

> 设计取舍：用哈希而非"从原点累加的 RNG 流"，是为了能在**任意 `t` 直接 O(1) 求值**，无需从序列起点重放。

## 3. 相干噪声：平滑、缓慢、有波动性

### 3.1 锚点 + 平滑插值（value noise）

只在间隔为 `period` 的锚点上取随机值，锚点之间用 `smoothstep` 插值：

```python
def smoothstep(x: float) -> float:
    return x * x * (3.0 - 2.0 * x)        # C1 连续，端点导数为 0，无折角

def coherent_noise(key: str, t_min: float, period_min: float, channel: str = "") -> float:
    """t_min、period_min 以分钟为单位。返回平滑的 [-1,1] 噪声。"""
    pos  = t_min / period_min
    i    = int(pos // 1)                   # 左锚点索引
    frac = pos - i
    a    = rand_unit(key, i,     channel)
    b    = rand_unit(key, i + 1, channel)
    return a + (b - a) * smoothstep(frac)  # 平滑插值
```

- `period`（锚点间隔）决定"波动有多慢"：取 6 小时 → 缓慢主起伏；取 30 分钟 → 起伏较快。**这正是"缓慢增长的波动"的控制旋钮。**
- 任意 `t` 只看左右两个锚点，**O(1)**，处处平滑、可复现。

### 3.2 分形叠加（fBm）：多尺度波动性

单一频率太规则。把多个频率（倍频/octave）叠加，得到"大尺度缓慢漂移 + 小尺度细节抖动"的自然质感：

```python
def fbm(key: str, t_min: float, base_period_min: float = 360.0,
        octaves: int = 4, lacunarity: float = 2.0, gain: float = 0.5,
        channel: str = "") -> float:
    """分形布朗运动式噪声，归一到 [-1, 1]。"""
    total = 0.0; amp = 1.0; period = base_period_min; norm = 0.0
    for o in range(octaves):
        total += amp * coherent_noise(key, t_min, period, f"{channel}#oct{o}")
        norm  += amp
        amp    *= gain                      # 高频幅度递减
        period /= lacunarity                # 频率翻倍
    return total / norm                     # 归一
```

参数语义速查：

| 参数 | 含义 | 调大的效果 |
|---|---|---|
| `base_period` | 最低频（最慢）成分的周期 | 主起伏更缓慢 |
| `octaves` | 叠加的频率层数 | 细节更丰富 |
| `gain` | 相邻倍频的幅度衰减 | 大→高频更明显（更毛糙） |
| `lacunarity` | 相邻倍频的频率倍数 | 一般固定为 2 |

## 4. 规则原语库

每条业务规则是引擎里的一个 `kind`，本质都是"包络（确定性趋势）+ 相干噪声"的组合。下面给出文档涉及的全部原语。

### 4.0 原语 / kind 速查表（开发与配置的唯一对照）

**需要实现为独立生成函数的 `kind`（`generators/` 注册，config `rule.kind` 引用）：**

| 原语（脾气） | `kind`（注册名） | 量语义 | 关键参数 | 客户指标示例 | 详见 |
|---|---|---|---|---|---|
| 上下飘·均值回归 | `有界瞬时` | 瞬时 | 基线, 幅度, base_period, octaves, 季节{振幅,周期,相位}, 区间 | 温度/航速/转速/压力/PH/浓度/真空度/湿度/得率读数（占多数） | §4.1–4.2 |
| 只增不减·计数器 | `累计` | 累计 | 速率基线, 漂移幅度, base_period, 起点 | 累计捕捞量/各线产量/各类消耗/耗油耗电/库存 | §4.3 |
| 跳挡·离散枚举 | `离散状态` | 瞬时(枚举) | 驻留分钟, 取值[], 权重[] | 设备状态/拖网绞车状态/速冻机开停 | §4.6 |
| 跟着别人算·派生 | `航迹`（位置→航向航速） | 瞬时(派生) | 航点[], 漂移, base_period, Δ | 航向/航速（从经纬度派生） | §4.5 |

**不是独立 `kind`，开发时按下列归属处理（避免误建函数）：**

| 项 | 实际归属 | 说明 |
|---|---|---|
| 趋势 / 季节包络（§4.2） | `有界瞬时`/`累计` 的**参数** | 通过 `季节{}`、线性/logistic 包络参数表达，非独立函数 |
| 多序列（§4.4） | `有界瞬时` + 不同 `channel` | 同一 kind 换子种子，非独立函数 |
| 瞬时速率 / 窗口聚合 | `累计` 的**差分**（resolver 求） | `r(t)=C(t)−C(t−Δ)`、窗口=末−首，不单独配置 |
| 指标间相关（§4.7，软关联） | **共享潜变量**（参数层） | 多指标引用同一 `env` 子种子，非 kind |
| 跨指标硬约束 / 派生（§5） | `constraints.yaml`（取数契约 §3） | 物料守恒/温度上界等跨 key，落约束层，非 kind |
| 固定值 / 冻结 | `人工固定` / `overrides.yaml` | 配置即数据，不进引擎 |

> 据此，**待开发的生成函数只有 4 个 `kind`**（`有界瞬时`/`累计`/`离散状态`/`航迹`）+ 包络/channel 参数 + 约束层；其余皆为这 4 个的参数化或组合。新增数据形态时，先判断它属于上表哪一行，仅当确实是新"脾气"才新增 `kind`。

### 4.1 有界瞬时量（温度 / 航速 / 得率 / 残留率）

```python
def 有界瞬时(p, key, t_min):
    season = p.季节振幅 * sin(2*pi * (t_min/p.季节周期_min) + p.季节相位)
    v = p.基线 + p.幅度 * fbm(key, t_min, p.base_period, p.octaves) + season
    return clamp(v, p.下界, p.上界)
```

示例（海水温度 25±2、缓慢起伏、年季节）：

```yaml
rule:
  kind: 有界瞬时
  基线: 25
  幅度: 2                  # → 落在 ±2
  base_period: 360         # 6h，主起伏缓慢
  octaves: 4
  季节: { 振幅: 8, 周期: 年, 相位: 0 }   # 冬冷夏热
  区间: [-2, 30]
```

> "25±2 且缓慢波动"被拆成：基线=25、幅度=2、base_period=慢、季节=年周期。每个语义都是一个显式参数。

### 4.2 趋势 / 季节包络

包络是叠加在噪声上的**确定性闭式函数**，与噪声相加：

- 线性：`g(t) = k·elapsed`
- logistic（缓慢饱和增长）：`g(t) = L / (1 + e^{-k(t - t0)})`
- 季节：`g(t) = A·sin(2π·t/T + φ)`

"缓慢增长"既可用低频 fBm（无方向的慢漂移），也可叠加线性/logistic 包络（有方向的慢增长），二者可并存。

### 4.3 累计量（累计捕捞 / 累计耗油 / 库存）

直接定义**单调的闭式累计曲线** `C(t)`，而非把增量累加 N 次：

```python
def 累计(p, key, t_min):
    elapsed = t_min - origin_min(p.起点, t_min)        # 从渔季/年初起的分钟数
    drift   = p.漂移幅度 * fbm(key, t_min, p.base_period, p.octaves)
    return p.速率基线 * elapsed + drift                # 主体单调 + 缓慢漂移
```

- **单调性保证**：只要 `速率基线 > 漂移幅度 / base_period`（漂移的最大斜率），`C(t)` 处处递增。这是一个可校验的参数约束。
- **瞬时速率派生**：`r(t) = C(t) − C(t−Δ)`，与累计曲线天然一致。
- **窗口聚合**：当天/当月/当前渔季/年度 = `C(end) − C(start)`，**O(1)**、精确、且与瞬时序列互相自洽。

### 4.4 多序列（拖网捕捞 / 泵吸捕捞 等）

同一图表的多条线，用不同 `channel` 子种子，各自独立平滑：

```python
拖网 = 有界瞬时(p_拖网, key, t_min)   # channel 隐含在 key 或参数里
泵吸 = 有界瞬时(p_泵吸, key, t_min)   # 用不同子种子 → 互不相关
```

### 4.5 航迹（经纬度 + 航向 + 航速）：派生一致性

先定义平滑的**位置函数**，航向、航速由位置的有限差分**派生**，从而三者物理自洽：

```python
def 位置(p, key, t_min):
    base = 航线插值(p.航点, t_min)                     # 预设航线上的点（缓慢推进）
    d经  = p.漂移 * fbm(key, t_min, p.base_period, channel="lon")
    d纬  = p.漂移 * fbm(key, t_min, p.base_period, channel="lat")
    return (base.经度 + d经, base.纬度 + d纬)

def 航迹(p, key, t_min, Δ=180):    # Δ=3min
    经,纬   = 位置(p, key, t_min)
    经0,纬0 = 位置(p, key, t_min - Δ)
    航速 = haversine(经0,纬0, 经,纬) / (Δ/60) * 节换算
    航向 = bearing(经0,纬0, 经,纬)
    return { 经度:经, 纬度:纬, 航速:航速, 航向:航向 }
```

> 这样绝不会出现"坐标不动却航速很大"的穿帮——因为航速就是从坐标算出来的。

### 4.6 离散状态（设备状态 / 拖网绞车状态）

按"驻留块"取值：每个时段块一个状态，块内恒定（不抖动），块界切换：

```python
def 离散状态(p, key, t_min):
    block = int(t_min // p.驻留分钟)
    u = (rand_unit(key, block, "state") + 1) / 2        # → [0,1)
    return 按权重映射(u, p.取值, p.权重)                  # 落到某个枚举
```

- 块内恒定 → 不像白噪声那样闪烁；块界切换 → 像真实状态变迁；**O(1)、可复现**。
- 需要更真实的转移（避免等概率乱跳）时，可让块 `i` 的状态以块 `i−1` 为条件做马尔可夫转移：查询区间内按块前向迭代即可（区间内块数很少，开销可忽略）；默认采用独立块方案，避免跨查询的历史依赖。

### 4.7 指标间相关（风大 → 浪高）

让相关的指标共享一个**潜变量**，再各自叠加特异噪声：

```python
env = fbm("环境.海况", t_min, base_period=720)   # 共享海况潜变量
风速 = 8 + 4*env + 1.0*fbm(key, t_min, channel="风速特异")
浪高 = max(0, 1 + 2*env + 0.3*fbm(key, t_min, channel="浪高特异"))
# 二者都跟随 env → 自动正相关
```

## 5. 派生一致性与指标间约束

### 5.1 派生一致性原则

引擎的一条贯穿性设计原则：**凡是物理上应当联动的量，都从同一个底层函数派生，绝不各摇各的。**

| 联动关系 | 派生方式 |
|---|---|
| 累计 / 瞬时速率 / 窗口聚合 | 全部由同一条 `C(t)` 求值或差分 |
| 坐标 / 航向 / 航速 | 全部由同一个位置函数 `pos(t)` 派生 |
| 风速 / 浪高 等环境量 | 共享潜变量 `env(t)` |
| 当天 / 当月 / 当前渔季 / 年度 | 同一 `C(t)` 在不同窗口端点求差 |

这同时带来"逼真"（量与量之间关系正确）与"自洽"（前端拿到的数据互不矛盾）。

### 5.2 指标间约束（跨指标派生）

上一节是"单指标内部"的一致性；同样的思路可推广到**指标之间的硬约束**，例如"冻虾舱温度 ≤ 海水温度""累计捕捞量 ≥ Σ 各产线原料流量"。

**核心命题：保证约束的唯一可靠方式是"按构造派生"，而不是"各自独立生成然后祈祷恰好满足"。** 懒计算天然支持——所有指标都是"同一个 `t` 的纯函数"，纯函数之间可组合。

**机制：指标依赖图（Metric DAG）+ 同一 t 求值。** 允许一个指标的规则引用其它指标；求值 `X(t)` 时在**同一个 `t`** 递归求出被引用指标再做运算，形成一张**有向无环图**：

- 仍是纯函数：`X(t)` 只通过依赖间接依赖 `t` → 依旧可复现、O(小)。
- 同一次请求内对 `(key,t)` 做记忆化，共享依赖（如海水温度被多处引用）只算一次。
- 不变式：图必须**无环**（循环依赖非法，加载期校验）。

**两种派生策略：**

| 策略 | 做法 | 特点 |
|---|---|---|
| A 结构性派生（推荐） | 受约束量定义为基准量的代数表达式，使约束恒等成立 | 精确、信号自然 |
| B 投影 / 钳制 | 照常独立生成，求值时对基准量 `min/max` 截断 | 简单，但贴边会被压平失真 |

**例 1 · 温度上界（冻虾舱温度 ≤ 海水温度）：**

```python
海水温度(t)   = 有界瞬时(...)                              # 独立基准
冻虾舱温度(t) = min( 有界瞬时(基线=-25, ...), 海水温度(t) - δ )   # 策略B
# 或 策略A： 冻虾舱温度(t) = 海水温度(t) - gap(t),  gap(t) ≥ 安全间隔 > 0
```

**例 2 · 物料守恒（累计捕捞量 ≥ Σ 各产线原料流量）：**

```python
累计捕捞量 C(t)                                   # 独立驱动（累计型）
α_虾肉(t), α_虾粉(t), α_冻虾(t) ≥ 0,  Σα ≤ 1       # 平滑噪声归一化得到的分配因子
原料流量_虾肉(t) = α_虾肉(t) · C(t)
原料流量_虾粉(t) = α_虾粉(t) · C(t)
原料流量_冻虾(t) = α_冻虾(t) · C(t)
# 由构造： Σ原料流量 = (Σα)·C ≤ C = 累计捕捞量   ✓ 每个 t 恒成立
```

要点：**选一组"独立基准"（海水温度、累计捕捞量），其余受约束量全部派生出来**，约束就是代数恒等式而非碰运气。

**边界：** 依赖图必须无环（相互约束须指定谁基准谁派生）；策略 B 频繁触界会压平曲线，改用 A；软关联（相关而非硬不等式）用共享潜变量（4.7）即可，不必进 DAG；依赖外部独立录入事件的约束需把事件纳入配置。

## 6. 求值流程（Pipeline）

```
series 查询 (key, start, end)
        │
        ▼
  按指标 grid 枚举网格点 t_grid ∈ [start, end]
        │
        ▼  对每个 t_grid：
  ┌─────────────────────────────────────────────┐
  │ 1. override 命中? → 返回覆盖/冻结值           │
  │ 2. 否则 computed provider：                   │
  │    a. 取该指标 rule.kind                       │
  │    b. GENERATORS[kind](params, key, t_grid)   │
  │       └─ 内部：哈希播种 → 相干噪声/fBm         │
  │          → 叠加包络 → clamp / 派生              │
  └─────────────────────────────────────────────┘
        │
        ▼
  {time, value} 点 → 组装为 values 序列
```

全程**无读写、无状态、无外部 I/O**。点查/即时 = 单点求值；区间 = 多点求值；累计窗口 = 2 点求差。

## 8. 正确性与可复现性的验证方法

引擎应配套以下自动化校验（建议进单元测试）：

| 性质 | 校验方法 |
|---|---|
| **确定性** | 同 `(key,t)` 调用两次结果相等；跨进程重启与"黄金样本"文件比对一致 |
| **值域** | 采样 N 点，断言全部 ∈ `[下界,上界]`、均值 ≈ 基线、标准差 ≈ 期望 |
| **平滑性** | 相邻网格点差值 `max|v(t+Δ)−v(t)|` 低于阈值（排除白噪声跳变） |
| **缓慢性** | 自相关在 `base_period` 量级仍显著（确认起伏确实"慢"） |
| **累计单调** | `C(t)` 非递减；`C(end)−C(start)` 等于逐网格增量之和（抽样核对） |
| **派生一致** | 由坐标差分算出的航速/航向 与 `航迹()` 返回值一致 |
| **相关性** | 共享潜变量的两指标相关系数符号/量级符合预期 |

## 9. 性能

每个点的成本 = `octaves` 次哈希 + 常数次乘加。

| 查询 | 点数 | 量级 |
|---|---|---|
| 点查 / 即时 | 1 | 微秒级 |
| 当天（3min 网格） | ~480 | < 1 ms |
| 半年趋势（3min） | ~86k | 数十 ms |
| 累计窗口（当月/渔季/年度） | 2（求差） | 微秒级 |

### 9.1 实测（基准）

单点成本随哈希实现差一个量级，下表为「近一年逐小时 = 8760 点/指标」的实测：

| 实现 | 单点 | N=1 | N=10 | N=100（87.6 万点） |
|---|---|---|---|---|
| 纯 Python `sha256`（本文原型） | ~4.5 µs | 41 ms | 382 ms | 3.9 s |
| 向量化 + 便宜哈希（xxhash/SplitMix64，§2 已留口） | ~0.05 µs | 1.6 ms | 4.6 ms | 46 ms |
| 对照：SQLite 建好库直接查 | — | 2.6 ms | 23 ms | 234 ms |

两点结论：

- **原型够用，优化后反超落库**：原型那段 `sha256` 每点 8 次加密哈希，是最坏情况；换成便宜哈希 + 按时间轴向量化后单点快约 90×，大查询（N=100）46 ms，**比 SQLite 读同样多行（234 ms）还快**——因为引擎全程内存算术，省掉了逐行反序列化与磁盘 I/O。故大查询走向量化路径，原型仅用于单点/小区间。
- **量大时瓶颈在序列化/传输，不在计算**：87.6 万点无论算还是查，JSON 编码与网络传输（数十 MB）都会盖过生成成本——这进一步说明该用降采样（见《配置与取数契约》§5.1），而非纠结生成方式。

结论：性能上**没有任何引入数据库或缓存的理由**；落库反而引入采集、回填、漂移、一致性等额外复杂度而无收益。落库唯一的性能优势出现在「同一段历史被反复查且生成本身很贵」时，那时给引擎加一层内存记忆化即可，仍不必引入数据库。

## 10. 边界与不适用

- **无界布朗运动**（方差随时间无限增长）本引擎不模拟。但温度、航速、得率等是**有界、均值回归**的量，恰好适合相干噪声；累计量是单调的，用闭式趋势——文档中的量全部落在这两类。
- **改了 `rule` 参数 → 该指标历史随之重算**（这是"规则即真相"的必然）。需要冻结某段历史，用 `overrides` 钉死。
- **真正外部独立录入、且路径依赖的事件流**（如逐笔出入库），纯函数算不出；但此类在本系统中作为维度/overrides 配置维护，仍可计算或直接读取。
- **本引擎仅服务 `computed` 指标**。工厂天气（open-meteo→SQLite）、船舶数据（TimescaleDB）是真实数据，走查询不走计算，其可复现性由库中真实记录保证，与本机制无关。

---

## 11. 小结

确定性造数引擎用两件简单的事，解决了"无库、可复现、又逼真"的三难：

1. **哈希播种的纯函数** → 可复现（同输入永远同输出）。
2. **相干噪声 + 分形叠加 + 派生一致性** → 逼真（平滑、缓慢、有波动性、量间自洽）。

业务"规则"被拆解为基线、幅度、周期、倍频、趋势包络、驻留、潜变量等显式参数，全部声明在配置里。这是图形学/程序化生成领域成熟的技术，迁移到时序数据 Mock 上，简单且可靠。
