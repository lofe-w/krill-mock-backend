## 1. 系统是什么

为"深蓝号"磷虾船 + 连云港工厂可视化大屏提供数据的后端。一句话模型：

> **一个配置驱动的 `key | filter | value` 查询服务 + 一个确定性造数引擎。**

数据按"值怎么来"分三类：**静态固定值**、**动态规则生成**、**特殊来源**。三类共用同一个查询接口 `/api/query`（特殊类按需另有少量专用接口）。

设计底线：简单、快、通用、可配置、**可复现**（同一查询任何时候结果一致）。

## 2. 统一查询接口 `/api/query`

### 2.1 契约

```
POST /api/query
Content-Type: application/json
```

**入参**

| 字段 | 类型 | 说明 |
|---|---|---|
| key | String | 选定模块/实体/功能，如 `溯源`、`仓库`、`船舶.冻虾舱` |
| filter | Object | k-v 过滤条件；`{}` 表示不过滤。支持单选/多选、字符串/数值/时间 |

**出参**

`data` **恒为数组**，每个元素是一条"命中行" `{ filter, value }`（`key` 已在请求中，省略）。命中 0/1/N 条形态一致，客户端无需判类型：

```jsonc
{
  "status": 200,
  "data": [
    { "filter": { /* 该命中行的标识，如 {"产品批号":"QZXF-001"} */ },
      "value":  { /* 业务数据，原样返回 */ } }
    /* ... 命中多条时多个元素；命中 0 条时 data 为 [] */
  ]
}
```

- 协议字段（`status`/`data`/`key`/`filter`/`value`）用英文；**`value` 内部及 `filter` 内的业务键用中文**（如 `值`/`单位`/`时间`/`类别`/`溯源链条`/`产品批号`）。
- `data` 为数组的原因：`filter` 可能命中 **0~N** 条；统一为数组避免"单条返 Object、多条返 Array"的类型漂移。元素自带 `filter` 以便区分（尤其多条、或动态点查时）。
- 元素中的 `filter` 是**命中行的标识**（如指标、产品批号等非时间维），不含查询时传入的时间。
- 后端按 `key` 在注册表中的**类别**自动分派（静态查配置 / 动态算规则 / 特殊走来源），调用方无需关心类别。

### 2.2 错误码

| status | 含义 |
|---|---|
| 200 | 成功（含命中 0 条：`data` 为 `[]`） |
| 400 | 参数错误（缺 key / filter 格式错） |
| 404 | `key` 不存在（未注册） |
| 500 | 服务端异常 |

> 区分：`key` 未注册 → 404；`key` 存在但 `filter` 命中 0 条 → 200 + `data: []`。

## 3. 三类数据详解

### 3.1 ① 静态固定值（人工配置）

人工在 YAML 配置的固定数据：溯源、仓库、监控地址、人员列表、船舶信息、专利、设备铭牌等。**配置内容即数据即出参。**

- `filter` 在该 `key` 的所有配置条目中筛选；`filter={}` 命中全部。每个命中条目 → `data` 数组中一个 `{filter, value}` 元素。
- `value` 是配置里写死的任意 JSON，原样返回。

**示例 · 溯源（按产品批号查，命中 1 条）**

```jsonc
// req
{ "key": "溯源", "filter": { "产品批号": "QZXF-001" } }
// resp
{ "status": 200, "data": [
  { "filter": { "产品批号": "QZXF-001" },
    "value": {
      "类别": "全脂虾粉溯源",
      "溯源链条": [
        { "磷虾捕捞":   { /* 过程参数 */ } },
        { "船载加工":   { /* 过程参数 */ } },
        { "冷链运输":   { /* 过程参数 */ } },
        { "岸基存储":   { /* 过程参数 */ } },
        { "全脂虾粉产品检测": { /* 过程参数 */ } }
      ]
    } }
] }
```

**示例 · 仓库（按仓库类别查）**

```jsonc
// req
{ "key": "仓库", "filter": { "仓库类别": "深蓝号冻虾舱" } }
// resp
{ "status": 200, "data": [
  { "filter": { "仓库类别": "深蓝号冻虾舱" },
    "value": { "批号": "LXY-001", "数量": 3000, "单位": "kg", /* ... */ } }
] }
```

**示例 · 命中多条（filter 为空 → 全部 6 个仓库）**

```jsonc
// req
{ "key": "仓库", "filter": {} }
// resp
{ "status": 200, "data": [
  { "filter": { "仓库类别": "深蓝号冻虾舱" }, "value": { /* ... */ } },
  { "filter": { "仓库类别": "深蓝号虾粉舱" }, "value": { /* ... */ } },
  /* ... 其余 4 个 */
] }
```

**示例 · 单例（filter 为空，命中 1 条）**

```jsonc
{ "key": "船舶.监控地址", "filter": {} }
// resp.data = [ { "filter": {}, "value": ["rtsp://camera1...", "rtsp://camera2..."] } ]

{ "key": "船舶.IMO", "filter": {} }
// resp.data = [ { "filter": {}, "value": "9849332" } ]
```

### 3.2 ② 动态规则生成（可复现）

按 YAML 规则即时生成、不落库、可查历史、稳定可复现：温度、浓度、产量、能耗、设备运行状态等。

- `key` = 模块/实体（如 `船舶.冻虾舱`）；**"指标"放在 `filter`**（如 `{"指标":"温度"}`）。
- **点查**：filter 带 `时间`（缺省则取"现在"）→ 返回 ≤ 该时刻的**最近网格点**的单值。
- **范围查**：filter 带 `开始时间` + `结束时间` → 返回区间内**全部网格点**序列。
- 网格由配置的 `cron` 决定（见 §4.2）；可复现性由造数引擎保证（见 §5）。

**点查**

```jsonc
// req
{ "key": "船舶.冻虾舱", "filter": { "指标": "温度", "时间": "2026-06-20 13:00:00" } }
// resp
{ "status": 200, "data": [
  { "filter": { "指标": "温度" }, "value": { "值": -11.2, "单位": "℃" } }
] }
```

**范围查**

```jsonc
// req
{ "key": "船舶.冻虾舱",
  "filter": { "指标": "温度", "开始时间": "2026-06-20 13:00:00", "结束时间": "2026-06-20 15:00:00" } }
// resp
{ "status": 200, "data": [
  { "filter": { "指标": "温度" },
    "value": {
      "单位": "℃",
      "值": [
        { "时间": "2026-06-20 13:00:00", "值": -11.2 },
        { "时间": "2026-06-20 14:00:00", "值": -10.8 },
        { "时间": "2026-06-20 15:00:00", "值": -11.0 }
      ]
    } }
] }
```

> 即时值：filter 不带任何时间 → 等价于点查"现在"，返回 ≤ now 的最近网格点。
> 多指标：filter 的 `指标` 多选或缺省（命中该 key 下多个指标）时，`data` 含多个元素，各元素 `filter` 标明指标。

### 3.3 ③ 特殊来源（按需支持）

需特殊支持的数据：直接接三方接口、接三方并落库、RTSP 视频流、调用 LLM 等。这类**不一定走 `/api/query`**，接口实现与数量视情况而定（见 §6）。

## 4. 配置模型（YAML）

一切由配置驱动，**配置即生效**。注册表按 `key` 组织，每个 key 声明其类别与内容。

### 4.1 静态配置

配置内容 ≈ 数据模型本身：`key` 下若干条目，每条 `filter`（标识）+ `value`（原样数据）。

```yaml
- key: 溯源
  类别: 静态
  条目:
    - filter: { 产品批号: QZXF-001 }
      value: { 类别: 全脂虾粉溯源, 溯源链条: [ { 磷虾捕捞: {...} }, ... ] }
    - filter: { 产品批号: DX-001 }
      value: { 类别: 冻虾溯源, 溯源链条: [ ... ] }

- key: 船舶.IMO
  类别: 静态
  条目:
    - filter: {}
      value: "9849332"
```

### 4.2 动态配置

只配 `key` + `filter`（非时间维，如指标）+ 值类型 + 单位 + 网格(cron) + 规则及参数；**不配具体时间与值**。

```yaml
- key: 船舶.冻虾舱
  类别: 动态
  指标:
    - filter: { 指标: 温度 }
      值类型: Number
      单位: ℃
      cron: "0 * * * *"            # 每小时一个网格点
      规则: { kind: 有界瞬时, 基线: -10, 幅度: 2 }   # -10±2℃，随时间随机但历史可复现

- key: 船舶.虾粉舱
  类别: 动态
  指标:
    - filter: { 指标: 温度 }
      值类型: Number
      单位: ℃
      cron: "10 * * * *"
      规则: { kind: 有界瞬时, 季节: { 春: -10, 夏: -5, 秋冬: -15 }, 幅度: 2 }
```

> `cron` 表达"值存在的时间网格"（如每小时一个点），**并非真有定时任务在跑**——动态数据是懒计算的：点查时把请求时间对齐到网格、按规则即时算出值。

### 4.3 特殊配置

声明来源类型与连接信息（密钥经环境变量，不写进 YAML）。

```yaml
- key: 工厂.天气
  类别: 特殊
  来源: { kind: open-meteo, 落库: sqlite, 定位: { 纬度: 34.60, 经度: 119.22 }, 采集cron: "0 * * * *" }

- key: 船舶.航行
  类别: 特殊
  来源: { kind: timescaledb, 表: ship_metrics }      # 外部进程每10min写，本地只读透传

- key: 船舶.监控视频
  类别: 特殊
  来源: { kind: rtsp, 服务: mediamtx }                # 循环推流，返回 RTSP 地址
```

## 5. 动态数据生成原理（可复现）

> 仅列实现要点；完整算法、参数、验证见过程文档《造数引擎》。

### 5.1 可复现的根基

值 = `f(seed(key, 指标, 网格时间))` 的纯函数。随机性来自对 `(key, 指标, 网格时间)` 的哈希播种，**只依赖输入、不依赖 now / 调用次序 / 任何状态** → 同一时刻任何时候查结果一致。

### 5.2 逼真性

不能逐点独立摇随机数（那是乱跳的白噪声）。用**相干噪声**：只在稀疏锚点上取随机值、之间平滑插值，再叠加多倍频(fBm)与季节/趋势包络 → 平滑、缓慢起伏、有波动性。`幅度` 控制 ±range，`base_period` 控制起伏快慢。

### 5.3 预设规则目录（kind）

| kind | 用途 | 关键参数 |
|---|---|---|
| 有界瞬时 | 温度/浓度/得率等有界量 | 基线、幅度、季节、区间(clamp) |
| 累计 | 累计产量/耗油等单调量 | 速率基线、起点；窗口量 = C(end)−C(start) |
| 离散状态 | 设备状态/绞车状态等枚举 | 取值、权重、驻留时长 |
| 航迹 | 经纬度/航向/航速 | 航线、漂移；航向航速由坐标派生 |

新增一种数据形态 = 加一个 kind 函数 + 在配置引用。

### 5.4 指标间约束

硬约束（如"冻虾舱温度 ≤ 海水温度""累计捕捞量 ≥ Σ各产线原料流量"）用**指标依赖图 + 按构造派生**：受约束指标定义为基准指标的代数表达式（差值、份额、min/max），在同一时刻递归求值，约束恒成立。依赖图必须无环。

## 6. 特殊来源接入

| key（示例） | 来源 | 取数方式 | 历史 |
|---|---|---|---|
| `工厂.天气` | open-meteo | 定时采集真实数据写本地 SQLite，查询直读 SQLite | SQLite 累积 + 归档回填 |
| `船舶.航行`、`船舶.海况` | 远程 TimescaleDB | 只读透传（外部进程每 10min 写，本地不落库） | TSDB 自带 |
| `船舶.监控视频` / `工厂.监控视频` | RTSP（MediaMTX） | 循环推流，返回 `rtsp://本系统:8554/{监控ID}` 地址 | 不适用 |
| （预留）`*.LLM问答` 等 | LLM | 调用大模型 | 视情况 |

- 这些可走 `/api/query`（返回值/地址），视频流本体由播放器直连 RTSP，后端不经手字节。
- 真实来源不可用时可回落到造数兜底（按 key 配置）。

## 8. 通用约定

- **命名**：协议/结构字段用英文（`status`/`data`/`key`/`filter`）；业务名与 `value` 内部键用中文（`值`/`单位`/`时间`/`类别`/`溯源链条`…）。
- **时间格式**：`YYYY-MM-DD HH:mm:ss`；日期 `YYYY-MM-DD`。
- **filter 语义**：`{}`=不过滤（命中全部）；多键=与条件；支持单选/多选、字符串/数值/时间。
- **可复现**：动态数据在"规则不变"的前提下历史一致；需冻结某段历史用 override 配置。
- **关注点分离**：后端只给数据；单位拼接、图表样式归前端。
