## 0. 注册表条目 Schema（每个 key 的声明结构）

```yaml
- key:        <全限定名 实体[.子实体].指标>
  表:         A | B | C
  value:      <标量 | 记录{字段…} | 累计 | 枚举维度 | 有序数组[{环节,数据}] | 业务标识(批号)链接>
  成熟度:     真值采集 | 规则生成 | 人工固定
  filter:     [<维度>]            # 仅 B：开放实例集（如 产品批号/仓库类别）
  量语义:     瞬时 | 累计           # 仅 C
  网格:       <cron，预设待调>      # 仅 C
  单位/范围:  <来自甲方>
  不变式:     [<编号，见 §7>]
  出处:       甲方 <节号>
```

表速记（`domain.md` §2）：**A**`(key,value)` 单一站立 · **B**`(key,filter,value)` 可选站立 · **C**`(key,time,value)` 时序。（D 当前不实例化，见 domain §2.4。）

## 6. 溯源（表 B · value=`{类别, 溯源链条}`，数据归各环节）✅

> **以甲方 6.x 为权威**：溯源是环节链，**追溯/检验是链条内某环节的数据**，非顶层字段——磷虾捕捞环节带捕捞海域/方式/时间（即三方接口的"原料来源追溯"），产品检测环节带检验报告（即三方接口的"产品检验"）。三方接口把这两块拍平成顶层字段是其重构，不跟。

| key | filter | value | 出处 |
|---|---|---|---|
| `溯源` | 产品批号 | `{类别, 溯源链条[有序数组]}`；每元素 `{环节, 批次编号, ...本环节具体值}`（捕捞→海域/方式/时间；加工→流量/温度/产量；检测→检测摘要） | 甲方 6 |

环节链按产品类别决定（路径长短不同，**不为 7 类各写一套**）：

| 产品类别 | 溯源链环节序列 |
|---|---|
| 冻虾 / 虾肉 / 全脂虾粉 | 磷虾捕捞 → 船载加工 → 冷链运输 → 岸基存储 → 产品检测 |
| 虾油 / 脱脂虾粉 | …岸基存储 → 虾油提取 → 产品检测 |
| 虾油胶囊 | …岸基存储 → 虾油提取 → 胶囊封装 → 产品检测 |
| 蛋白肽 | …岸基存储 → 虾油提取 → 蛋白肽加工 → 产品检测 |

**有序数组 + 具体快照值**：
- `溯源链条` 是**数组**（事件顺序，JSON 对象键序不可靠）；每元素 `{环节, 批次编号, ...具体值}`。
- 每元素 = **`{环节: <名>, 数据: {...}}`**：环节是显式字段(标签、不放进键)，数据是该环节具体值——标签与数据分层、schema 统一、可均匀遍历。
- 环节数据是**具体历史快照值**（人工固定，照甲方 6.x），**不是内部 key 引用**——溯源快照与实时流是不同事实，不存在重复。（早期误用"内部 key 引用"导致接口原样吐内部 key，已纠正、并废弃整个引用机制。）
- **批次编号是业务标识**，保留（虾油/蛋白肽的捕捞环节带 `原料虾粉/虾油批次` 可再追溯上游）；不暴露内部 key。
- **避免"同一事实两份"靠**：①同一事实只建一个 key（车间外大气温度=工厂天气，只建一处、前端直接查）；②跨实体用业务标识(批号)链接、各查各接口。均不需"引用"机制。

> 现状：Fake 阶段配置喂溯源，**配置结构 = 回溯产出结构**（`{类别, 溯源链条[有序]}`，环节自带具体值），保证将来切真实回溯时前端零改造。

## 7. 业务不变式 → key 映射（一致性契约）

> 仅取大屏看得见、观众能当场抓矛盾的（`domain.md` §6）。实现手段见《造数引擎》。

| 编号 | 不变式 | 涉及 key | 实现要点 |

| INV-1 | 物料守恒：输出 ≤ 输入 | `船舶.捕捞系统.累计产量.*` ≥ Σ`船舶.<生产线>.流量秤.当天累计`；虾油线消耗↔产出（`工厂.虾油提取生产线.生产数据.*`） | 选独立基准，其余按构造派生 |
