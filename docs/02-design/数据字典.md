## 0. 注册表条目 Schema（每个 key 的声明结构）

```yaml
- key:        <全限定名 实体[.子实体].指标>
  形状:       A | B | C | D
  value:      <标量 | 记录{字段…} | 枚举[…] | 累计 | 引用→key | 事件包 | 投影>
  成熟度:     真值采集 | 规则生成 | 人工固定
  filter:     [<维度>]            # 仅 B/D：开放实例集维度（如 产品批号）
  量语义:     瞬时 | 累计           # 仅 C/D 采样型
  网格:       <cron，预设待调>      # 仅 C/D
  单位/范围:  <来自甲方>
  不变式:     [<编号，见 §9>]
  出处:       甲方 <节号>
```

形状速记（`domain.md` §2）：**A**`(key,value)` 单一站立 · **B**`(key,filter,value)` 可选站立 · **C**`(key,time,value)` 单一时序 · **D**`(key,filter,time,value)` 可选时序。

图例：✅=本文已决策　❓=须业务/前端确认（汇总见 §11）

## 9. 业务不变式 → key 映射（一致性契约）

> 仅取大屏看得见、观众能当场抓矛盾的（`domain.md` §6）。实现手段见《造数引擎》。

| 编号 | 不变式 | 涉及 key | 实现要点 |
|---|---|---|---|
| INV-1 | 物料守恒：输出 ≤ 输入 | `船舶.捕捞.累计产量` ≥ Σ`船舶.<线>.流量秤.当天累计`；`生产作业.*` 事件包 inputs ≥ outputs；`工厂.*.生产数据` 消耗↔产出 | 选独立基准，其余按构造派生 |
| INV-2 | 得率 = 输出/投入 ∈(0,1]，派生量 | `船舶.虾肉线.得率`, `工厂.虾油线.生产数据.虾油得率`, `工厂.蛋白肽线.生产数据.得率` | 不独立取值，由 INV-1 两端算 |
| INV-3 | 谱系无环可达 | `溯源` 投影、`生产作业.*` | 沿成品→原料回溯；**回收无水酒精按新批次入账、不回指上游**（防环）✅ |
| INV-4 | 冷链温度安全且时序单调因果 | `冷链温度`, `平板速冻机.出冻温度`, 检测`冻块中心温度` | 中心温度 ≤ 冷库温度；环节时间 捕捞≤加工≤冷链≤仓储≤检测 |
| INV-5 | 实际产量 ≤ 设计能力 | 各 `*.累计产量/产量` ≤ 对应 `*.设计能力` | clamp 上界 |
| INV-6 | 成分自洽（相邻环节同物成分连续） | `生产作业.*` 输入输出成分 ↔ `检测` | 共享潜变量/派生 |
