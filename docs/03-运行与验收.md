# 运行与验收

> 回答：现在是个什么服务、怎么启动、怎么验收数据、接口在哪。

## 1. 这是个什么服务

一个**无状态的 HTTP 查询服务**（FastAPI/uvicorn）。它把"三表模型"对外投影成**一个统一查询接口**：

| 端点 | 作用 |
|---|---|
| `POST /api/query` | **主入口**：`{key, filter?, time?, start?, end?}` → 数据。按表(A/B/C)自然分派 |
| `GET /api/health` | 健康 + **静态自检**（key 数、表/成熟度分布、引用是否完整） |
| `GET /api/keys` | 列出全部已注册 key（挑着查时用） |
| `POST /api/reload` | 改完 `config/*.yaml` 热加载，无需重启 |
| `GET /docs` | **Swagger 交互界面**（最省事的验收方式） |

> **关于"接口还没设计"**：对齐前端那六个胖接口的**最终形态**确实故意压后（domain §8，接口是最上层、最可替换的一层）。但上面这个**统一查询接口已经实现并可用**——它就是三表的对外投影，**你现在就用它验收数据**。等前端需求定型，再加兼容层/调形态，底层数据与 resolver 不变。

## 2. 怎么启动（Docker Compose）

```bash
docker compose up -d --build      # 构建并后台启动
docker compose logs -f api        # 看日志
# 服务起在 http://localhost:8000
docker compose down               # 停
```

改了 `config/*.yaml` 后（compose 已把 config 挂进容器）：
```bash
curl -X POST http://localhost:8000/api/reload   # 热加载 + 重新自检
```

> 不想用 Docker 也可本机跑：`pip install -r requirements.txt && uvicorn app.api:app --port 8000`。

## 3. 怎么验收数据（三种方式，任选）

**方式一·最直观**：浏览器开 `http://localhost:8000/docs`，在 Swagger 里点 `POST /api/query`，填下面"验收清单"里的 body，点 Execute 看返回。

**方式二·一键全跑**：
```bash
bash examples/queries.sh          # 逐条打 13 个代表用例，肉眼核对
```

**方式三·先看自检**：`curl http://localhost:8000/api/health` → 应 `selfcheck_ok: true`、`keys` ≈ 58、`by_table` A13/B8/C37。自检过 = 配置无断链。

## 4. 验收清单（逐条核对）

| # | 查什么（POST /api/query 的 body） | 预期 / 核对点 |
|---|---|---|
| 1 | `{"key":"船舶.信息"}` | A 表；返回 `{IMO:"9849332", 船名:深蓝号}` |
| 2 | `{"key":"溯源","filter":{"商品编号":"XY-001"}}` | B；value=`{类别,溯源链条}`；链条含**虾油提取**环节；各环节是**批次编号+引用**（不内联过程数据） |
| 3 ★ | `{"key":"检测","filter":{"产品":"虾油","批次":"XYTQ-2026-001"}}` | **金蝶真实标定**：磷脂≈60.7% / EPA≈16.2 / DHA≈7.58 / 虾青素**181mg/kg** / 酸价**7.8** / 过氧化值0.03。← 拿这条核对"逼真度" |
| 4 | `{"key":"检测","filter":{"产品":"脱脂虾粉","批次":"TZXF-001"}}` | 真实标定：脂肪3.2 / 蛋白76.6 / 水3.2 / 盐2.76 / 灰11.5 |
| 5 | `{"key":"仓储","filter":{"仓库类别":"工厂冷库"}}` | B 多条；全脂虾粉 + 磷虾油两条 |
| 6 | `{"key":"人员","filter":{"所属":"磷虾船"}}` | B；4 人；职务/位置/在岗是**内嵌时序数组** |
| 7 | `{"key":"船舶.冻虾舱.温度","time":"2026-06-18 12:00:00"}` | C 点查；单值，**落在 [-25,-15]，约 -18** |
| 8 | 同 7 再查一次 | **值完全相同**（确定性/可复现 = 演示/截图/对账的基础） |
| 9 | `{"key":"船舶.冻虾舱.温度","start":"2026-06-18 00:00:00","end":"2026-06-18 06:00:00"}` | C 区间；**7 个网格点**，相邻平滑不乱跳 |
| 10 | `{"key":"工厂.虾油线.生产数据","filter":{"指标":"成品油产量"},"start":"...00:00:00","end":"...12:00:00"}` | C 累计；**单调递增**；速率按真实出油率18%标定 |
| 11 | `{"key":"船舶.桁杆泵吸系统.拖网绞车.状态","time":"..."}` | C 离散；值 ∈ {运行,待机,停止} |
| 12 | `{"key":"船舶.航行","time":"..."}` | 航迹；返回 {经度,纬度,航速,航向}，四量物理自洽 |
| 13 | `{"key":"不存在.key"}` | `status:404` |

**两个"逼真度"重点核对**（金蝶真实标定，见 config/README "数据真实度标定"）：
- 第 3/4 条的**成分质检值**——magnitude 应贴合真实磷虾油厂（磷脂~60、EPA~16、出油率18%）。
- 反过来：**工艺过程参数**（提取温度/乙醇浓度/MVR真空度）、**蛋白肽线**、**船载三线温度**、**捕捞参数**是**合理虚构**（金蝶无真实源），不要拿它们当真实数据验收——这条边界很重要。

## 5. 当前能验收到哪、还差什么

- ✅ 可验收：A/B/C 三表全部 key 的取值；确定性、区间、累计单调、离散、航迹；溯源 value 结构；金蝶真实标定的成分值。
- ⏳ 暂返回占位 note（下一步铺开）：`派生`指标（得率/百分比，=产出/投入，由 constraints 求）、`引用`字段的展开求值、多指标批量。
- ⏳ 待真实源（Q3/Q5）：天气/航行海况/监控当前用 Fake 占位（标 `目标成熟度:真值采集`）。
- ⏳ 接口最终形态：待前端需求定型后定稿（现统一查询接口已够验收）。
