# 运行与验收

> 现在是个什么服务、怎么启动、怎么验收数据、接口在哪。

## 1. 这是个什么服务

一个**无状态 HTTP 查询服务**（FastAPI/uvicorn）。接口 = **三表的外部投影，每表一个、各自批量**（domain §8：统一在 resolver，分形状在接口，不把三种形状塞一个信封）：

| 端点 | 表 | 入参 | 出参 |
|---|---|---|---|
| `POST /api/value` | A | `{keys:[...]}` | `{key: 值}` |
| `POST /api/records` | B | `{keys:[...], filter:{key:{...}}}` | `{key: [{filter,value}]}` |
| `POST /api/series` | C | `{keys:[...], time | start/end, 指标?}` | `{key: 序列 / 多指标}` |
| `GET /api/health` | — | — | 自检：key 数、表/成熟度分布、派生条数、引用是否完整 |
| `GET /api/keys` | — | — | 列出全部 key（挑着查时用） |
| `POST /api/reload` | — | — | 改完 config 热加载 + 重新自检 |
| `GET /docs` | — | — | **Swagger 交互界面（最省事的验收方式）** |

> **为什么三个接口而不是一个**：A/B/C 是三种不同形状（无time/记录/时序），塞进一个信封会让前端每次还得判断"这次是哪种形状"——等于在接口层假装只有一种形状，违背三表模型。拆开后**每个接口内部形状统一、天然支持批量**：N 个时序一次 `/series`、X 个静态一次 `/value`。底层仍是**一个 resolver**，三个接口只是薄壳。
>
> **接口最终形态**（对齐前端"按屏取一大包"的六个胖接口）仍按 domain §8 压后——那是 value/records/series 之上的**组合/兼容层**，待前端需求定型再加。现这三个接口已够验收。

## 2. 怎么启动（Docker Compose）

```bash
docker compose up -d --build      # 构建并后台启动，服务在 http://localhost:8000
docker compose logs -f api
docker compose down
```
改了 `config/*.yaml` 后（已挂载进容器）：`curl -X POST http://localhost:8000/api/reload` 热加载。
不用 Docker：`pip install -r requirements.txt && uvicorn app.api:app --port 8000`。

## 3. 怎么验收（三种方式）

1. **Swagger**：开 `http://localhost:8000/docs`，点对应接口填 body、Execute。
2. **一键全跑**：`bash examples/queries.sh`（逐条打 value/records/series 代表用例）。
3. **先看自检**：`curl localhost:8000/api/health` → 应 `selfcheck_ok:true`、`keys`≈58、`by_table` A13/B8/C37、`derivations`=5。

## 4. 验收清单（逐条核对）

**A · `/api/value`**
| body | 预期 |
|---|---|
| `{"keys":["船舶.信息","工厂.虾油线.设计能力"]}` | 两个值；IMO=9849332；虾油线日处理700吨 |

**B · `/api/records`**
| body | 预期 |
|---|---|
| `{"keys":["溯源"],"filter":{"溯源":{"商品编号":"XY-001"}}}` | value=`{类别,溯源链条}`；链条含**虾油提取**环节；环节是批次编号+引用（不内联） |
| ★`{"keys":["检测"],"filter":{"检测":{"产品":"虾油","批次":"XYTQ-2026-001"}}}` | **金蝶真实标定**：磷脂60.7/EPA16.2/DHA7.58/虾青素**181mg/kg**/酸价**7.8** |
| `{"keys":["仓储","人员"],"filter":{"仓储":{"仓库类别":"工厂冷库"},"人员":{"所属":"磷虾船"}}}` | 仓储2条；人员4人（职务/在岗为内嵌时序） |

**C · `/api/series`**
| body | 预期 |
|---|---|
| `{"keys":["船舶.冻虾舱.温度"],"time":"2026-06-18 12:00:00"}` | 单值，落 [-25,-15]，约 -18 |
| 同上再查一次 | **值完全相同**（确定性/可复现） |
| `{"keys":["船舶.冻虾舱.温度"],"start":"...00:00:00","end":"...06:00:00"}` | **7 点**序列，平滑 |
| ★`{"keys":["工厂.虾油线.生产数据"],"指标":"虾油得率","time":"..."}` | **派生≈18%**（=成品油/虾粉，金蝶真实出油率） |
| `{"keys":["船舶.能耗.剩余燃油百分比"],"time":"..."}` | **派生**≈剩余燃油/1000*100 |
| `{"keys":["工厂.虾油线.生产数据"],"指标":"车间外大气.温度","time":"..."}` | **引用**=工厂.天气.温度（同值，不复制） |
| `{"keys":["船舶.桁杆泵吸系统.拖网绞车.状态"],"time":"..."}` | 离散 ∈ {运行,待机,停止} |
| `{"keys":["船舶.航行"],"time":"..."}` | 航迹 {经度,纬度,航速,航向} 自洽 |
| 用错接口（A 表打到 /series） | 返回 `error` 提示用 /value |

## 5. 逼真度边界（务必区分，详见 config/README）
- **可当真实验收**（金蝶真实标定）：成品磷虾油/脱脂虾粉/全脂虾粉成分、出油率18%、批次/保质期/库存量级。
- **合理虚构，别当真实**：工艺过程参数（提取温度/乙醇/真空度）、蛋白肽线、船载三线温度、捕捞参数——金蝶无真实源。
